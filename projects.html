Projet 1: MauritUrgence - Application de Gestion de la Protection Civile

Projet 1 Fin d'Études
Contexte : Projet de fin d'études - ISCAE Nouakchott, Mauritanie
Équipe : 3 développeurs (Projet collaboratif)
Durée : 6 mois (2022)

Solution complète visant à améliorer la communication entre les citoyens et les services d'urgence en Mauritanie. L'application permet de signaler les dangers avec précision et fiabilité, réduisant les délais d'intervention des services de secours.

Fonctionnalités principales :

  • 📱 Application mobile pour citoyens et agents (Flutter)
  • 💻 Interface web d'administration (Laravel)
  • 📸 Signalement d'urgence avec géolocalisation et photos
  • 👥 Gestion des ressources (agents, véhicules, plannings)
  • 🔔 Système de notifications en temps réel
  • 📊 Traitement et suivi des interventions

Technologies utilisées :

Mobile :
Flutter Dart
Backend :
Laravel PHP
Frontend Web :
HTML5 CSS3 JavaScript
Base de données :
MySQL PhpMyAdmin
Méthodologie :
UML

Architecture du système :

📱 Application Mobile
  • Interface citoyens (signalement sans authentification)
  • Interface agents (planning, profil, notifications)
💻 Plateforme Web
  • Administration (gestion des ressources)
  • Support technique (traitement des urgences)

  • ✅ Réduction significative des délais de signalement d'urgence
  • ✅ Amélioration de la précision des informations transmises
  • ✅ Centralisation efficace de la gestion des ressources
  • ✅ Solution adaptée au contexte mauritanien

Projet 3: Visual Org Structure - Visualisation de Structures Organisationnelles

Projet 3 Équipe 5 personnes
Contexte : Projet d'équipe - Développement produit logiciel
Équipe : 5 développeurs (2 Front-End, 2 Back-End, 1 Designer/Team Leader)
Mon rôle : Développeur Front-End (TypeScript/Interface utilisateur)
Méthodologie : Agile - Tableau Kanban

Développement d'un produit logiciel capable de générer des représentations visuelles des structures organisationnelles d'entreprise. Le projet visait à résoudre les limitations des systèmes existants comme 1C en offrant une gestion hiérarchique complexe (jusqu'à 5 niveaux) avec navigation intuitive.

Objectifs du projet :

  • 🏗️ Création de MVP (Minimum Viable Product)
  • 🗂️ Gestion de hiérarchies complexes (5 niveaux)
  • 💾 Sauvegarde en base de données
  • 🔍 Navigation aisée dans les structures changeantes
  • 📊 Visualisations interactives d'organigrammes
  • ⚡ Résolution des limitations des systèmes existants
